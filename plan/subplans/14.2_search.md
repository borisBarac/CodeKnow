## Task 3: Rewrite the `/v1/search` handler

**File:** `packages/codeknow-api/src/codeknow_api/app.py`

Replace lines 58–73 with:

```python
from codeknow.schemas import HybridSearchResponse
from codeknow_api.models import SearchRequest

@app.post("/v1/search", response_model=HybridSearchResponse)
async def search(body: SearchRequest) -> HybridSearchResponse:
    from codeknow.vector.multi_search import multi_graph_search

    result = await asyncio.to_thread(
        multi_graph_search,
        body.query,
        graph_base_dir=GRAPH_DIR,
        slugs=body.repos,
        n_results_per_graph=body.n_results_per_graph,
        total_limit=body.top_k,
        traversal_depth=body.traversal_depth,
        embed_config=app.state.embed_config,
        chroma_config=app.state.chroma_config,
    )
    return result
```

## Risk notes

- **Breaking change**: `query` is now required and non-empty. Previously `""` was accepted. Clients sending `{}` get 422.
- **`asyncio.to_thread` + ChromaDB**: ChromaDB uses `requests`/`urllib3` which is thread-safe.
- **`app.state` config reuse**: `EmbeddingConfig` and `ChromaConfig` are simple settings objects — safe to share across requests. Heavy objects (`ChromaStore`, embeddings model) are still created per-call inside `multi_graph_search`.
